# Guide Website - 个人云原生网关控制台交接文档 (Human Readable)

## 1. 项目概述

本项目是一个高度现代化、具备惊艳视觉表现力的**个人私有云/Homelab 服务导航网关（Dashboard）**。
有别于传统的静态导航页，本项目主打**“动态、流体、极客与优雅”**的融合体验。其最大特色在于深度集成了 **Material Design 3 (Material You)** 的色彩提取引擎，能够根据用户上传的壁纸（图片或视频）动态生成全局 UI 的色彩主题，并实现了深色/浅色模式的完美适配。在最新的开发中，项目进一步完善了全局设置面板与完全可交互的“编辑模式（Edit Mode）”。

## 2. 技术栈与框架选型

*   **核心框架**：React 18 + Vite (极速构建与热更新)
*   **样式方案**：纯原生 Vanilla CSS + 全局 CSS Variables (完全摒弃重量级组件库，追求极致的定制自由度与玻璃拟物化质感，拒绝 Tailwind 的标签污染)
*   **拖拽排序**：`@dnd-kit/core` + `@dnd-kit/sortable` (用于取代传统的 react-beautiful-dnd，提供轻量且高度可控的流式网格拖拽排序能力)
*   **色彩引擎**：`@material/material-color-utilities` (Google 官方提供的 M3 HCT 色彩空间算法引擎，实现壁纸取色与动态调色盘生成)
*   **图标库**：`lucide-react` (轻量级、风格统一的精美 SVG 图标库)
*   **颜色拾取器**：`@jaames/iro` (高度可定制的色彩轮盘，用于手动调节基础色)
*   **数据持久化**：原生 `localStorage` 及 `localforage` (处理卡片、系统设置、媒体大文件的本地缓存机制)

## 3. 核心功能与架构设计

### 3.1 动态壁纸与色彩引擎 (Dynamic Theming)
*   **实现机制**：通过 `App.jsx` 读取媒体数据，利用 M3 算法提取主色调 (Base Color)，自动映射出多达 10 余种 UI 角色色（Primary, Surface, Outline, Container 等）。
*   **主题方案扩展**：支持切换不同的 Material 3 算法预设，如 `TONAL_SPOT`（柔和单色）、`VIBRANT`（高饱和鲜艳）、`EXPRESSIVE`（极具张力的色彩偏移）等。

### 3.2 布局与核心组件
*   **Hero Section**：页面顶部的个性化展示区。现已支持“大号悬浮卡片式 (Card)”与“极简无边界 (Minimal)”两种展示风格。用户可随时在后台切换。
*   **AppGrid 服务矩阵**：网格化的应用卡片展示区，具备极致流畅的 CSS 悬浮动效。通过 `dnd-kit` 实现了基于碰撞检测的二维网格拖拽功能。
*   **Uptime 联动视效**：卡片悬浮时触发 Uptime Kuma 风格状态轴，且可以通过后台单独开关“Uptime 监控悬浮动效”的渲染。
*   **社交媒体面板 (Social Settings)**：可增删改查页面头部的社交账号外链，面板进行了左右双栏响应式设计（PC/Pad横排，手机自动竖排）。

### 3.3 全局编辑模式 (Edit Mode)
系统不仅是展示页，更是灵活的管理后台：
*   **拖拽重排**：进入编辑模式后，卡片变成可抓取的幽灵状态（Dragging），支持自由排序。
*   **卡片修改与添加弹窗**：具备卡片增、删、改功能，支持丰富的 Lucide 图标可视化选择，纯正的 M3 拟物表单设计。
*   **全局设置 (Admin Modal)**：支持全局界面布局的调整、主题色彩的管理、社交媒体的修改等。

## 4. 技术细节与 UI/UX 亮点

*   **克制的极简动效 (Minimalist Animations)**：
    *   在开发中不断精简“缩放/弹跳 (scale/bounce)”等容易引起视觉疲劳和重绘卡顿的重度动画。
    *   模态框的出现、下拉框 (CustomSelect) 的展开、卡片的拖拽态均统一采用了**仅改变 opacity 与极微量 translateY 的纯浮现过渡 (FadeIn)**，杜绝了无谓的 `scale` 导致的不稳定感。
*   **中心化图标映射**：为了让配置数据被持久化保存，引入了 `iconMap.js` 与 `socialIconMap.js`，将 JSON 文本巧妙转换为 React 组件节点。
*   **完美对齐的间距哲学**：所有的表单内嵌卡片（如 Uptime 开关卡片）、自定义下拉框、社交卡片，其 padding、border-radius 均经过严苛的 1:1 像素级打磨以保持完全一致的 M3 设计语言。

## 5. 近期重大更新与重构 (Phase 2)

### 5.1 高性能动态取色引擎优化
针对用户上传 4K 高清原图或高码率视频作为壁纸时导致的浏览器主线程假死问题，进行了底层架构重构。
*   **实现机制**：在图像送入 `@material/material-color-utilities` 算法提取颜色前，引入离屏 `Canvas` 机制。将图片进行极限等比压缩，约束其最大边长不超过 128px。
*   **效果**：色彩提取消耗的像素样本从 800 万（4K）断崖式下降到 2 万以内，实现了任意大小壁纸取色的**毫秒级无感闪电完成**。

### 5.2 全局背景遮罩深度控制 (Advanced Overlay Controls)
将背景遮罩层的控制权彻底开放给用户，极大地提升了个性化自由度。
*   **功能项**：新增“开启背景遮罩”、“遮罩颜色跟随主题取色”以及“无级调节模糊度（Blur）”三大开关与滑条。
*   **实现细节**：相关状态在 `App.jsx` 层级进行统筹并下发给 `AdminModal`，所有调节参数实时转换为内联 CSS 变量与 `RGBA` 样式渲染到背景层，并持久化至 `localStorage`。

### 5.3 UI/UX 与交互细节极致打磨
*   **M3 极简滑块 (Android Style Slider)**：废弃了原有的廉价滑条样式，通过纯 CSS 精确重构了对标 Android 16 / M3 规范的系统级滑块（24px 粗轨道 + 12px 纯白拟物胶囊拨片），彻底杜绝了 WebKit 内核下因滥用 `overflow: hidden` 导致的滑块消失 Bug。
*   **悬浮动画防冲突解耦**：彻底解决了第一个卡片（或任意网格卡片）丢失悬浮上浮 `:hover` 动画的问题。通过在卡片外层单独包裹一层不可见的 `.sortable-wrapper` 作为拖拽底层坐标受体，完美避免了 `dnd-kit` 内联 `transform` 样式对 UI 视觉动画的“优先级霸凌”。

## 6. 近期重大更新与重构 (Phase 3 - 真云端持久化与视觉打磨)

### 6.1 Express 后端服务化与配置文件持久化
*   **架构设计**：新增 Express 服务端（`server.js`），运行于 `5000` 端口。前端通过 Vite 开发代理（`/api`）进行路由中转。
*   **真云端持久化**：所有系统设置、动态壁纸信息、头像信息、应用卡片矩阵以及社交媒体链接均在用户更改时自动通过防抖机制发送到 `/api/config` 写入服务端的 `data/config.json`，彻底告别了“换个设备或浏览器配置就丢失”的浏览器强绑定问题。
*   **本地兜底备份**：为保证极致的离线体验，即使后端服务关闭，前端也能无缝回退至 `localStorage` 和 `localforage` 缓存进行本地渲染与交互。

### 6.2 壁纸自动转码与静音自动播放机制 (FFmpeg 集成)
*   **自动转码 H.264**：针对用户上传壁纸时可能使用 Chrome 无法直接解码的 **H.265 (HEVC)** 编码格式视频的问题，后端服务内置了 **FFmpeg 自动转码引擎**。自动将所有上传的视频壁纸转换为全球设备 100% 兼容的 `H.264 (MP4)` 格式，确保在 Chrome、Safari 等浏览器上一致流畅展示。
*   **音频轨强制剥离（`-an`）**：在转码时自动切除音频轨道，不仅能大幅压缩约 30% 的文件体积，更完美顺应了所有浏览器对于**静音视频自动播放（Autoplay）**的严苛安全审查。
*   **前端播放优化**：针对 React 动态更新视频节点不触发 Autoplay 的缺陷，重构了 `App.jsx` 的渲染结构。为背景图/视频挂载了独有的 `key={mediaUrl}` 强制更新节点，配合 `defaultMuted` 和 React `ref` 强制将 DOM 元素 `muted` 属性赋值为 `true`，保障了全平台瞬间启动播放。

### 6.3 跨设备防浏览器强缓存机制
*   **服务器防缓存响应头**：在 `/api/config` GET 路由中追加了 `Cache-Control: no-store, no-cache, must-revalidate` 响应头，阻止代理和客户端强缓存旧配置文件。
*   **时间戳物理缓存穿透 (Cache Buster)**：在前端请求 `/api/config` 时追加动态时间戳 `?t=${Date.now()}`，彻底根治了 Chrome 浏览器因为本地 Disk/Memory Cache 缓存旧响应导致在其他浏览器上传壁纸后 Chrome 刷新不出来的 Bug。
*   **前端入口（index.html）缓存拦截**：在 `server.js` 静态资源以及 SPA 兜底服务中为 `index.html` 特殊拦截并设置了禁用缓存头，以保证前端更新代码发布后用户刷新可立刻读取最新的 JS/CSS 哈希文件名。

### 6.4 视觉与动效微调打磨
*   **膨胀与入场缩放动效优雅回归**：
    *   **应用卡片**：重新带回了用户最喜爱的悬浮微膨胀动效（`transform: translateY(-8px) scale(1.02)`），并将卡片进入页面（`cardEnter`）时的动画加入了 `scale` 从 `0.98` 到 `1` 的平滑过渡。同时全面保留了高端、非弹性的 **`cubic-bezier(0.25, 0.8, 0.25, 1)`（超平滑缓出）** 曲线，消除了廉价的 bouncy 回弹感。
    *   **个人主页卡片**：悬停时采用匹配的 `translateY(-6px) scale(1.01)` 连贯交互，且将卡片内悬浮图标缩放比提升至经典的 `1.1`。
*   **头像外轮廓主题色联动**：
    *   头像外圈从固定的黑白色改为 **`3px solid var(--primary)`**，其实时轮廓颜色将自动与壁纸中智能提取出的色彩主题进行高亮呼应，且在编辑状态下平滑变换为虚线。
*   **切换按钮完美视觉居中**：
    *   **清空默认 Padding**：清空了 `.btn-icon` 上的冗余外边距与内边距，完全隔离了网格卡片样式污染。
    *   **类名解耦**：将 TopBar 切换按钮内的动画容器类名重新重命名为专用的 **`.theme-icon-wrapper`**。
    *   **重归几何居中**：在完全重置按钮容器结构后，将 `.theme-toggle-btn svg` 的左偏移重归几何对称的 **`left: 0;`**，实现了太阳/月亮图标在正常、Hover、点击等任何交互下的像素级正中心对齐。
*   **色盘调节防卡顿重构**：
    *   针对原本色轮 `iro.ColorPicker` 随着 `baseColor` state 高频更新而不断销毁重建导致拖动卡顿、失焦的 Bug，将色轮的初始化彻底与 state 依赖解耦。色盘只在 Modal 打开时静态初始化一次，随后通过 programmatic 赋值（且带阈值过滤）单向同步 state 修改，实现了丝滑顺畅的手动拖拽改色体验。

## 7. 后续开发与交接注意事项

1.  **关于视频/头像文件物理持久化存储的部署**：
    当前服务端的上传文件托管在物理路径 `data/uploads/` 下，若在 Docker 或云端集群环境进行部署，**必须将 `data/` 目录挂载为持久化数据卷 (Volume)**，以防容器销毁时导致上传的自定义壁纸和头像文件丢失。
2.  **关于 FFmpeg 系统依赖**：
    后端转码极度依赖服务器本地的 `ffmpeg` 与 `ffprobe` 命令行工具。部署时需确保运行环境（或 Docker 基础镜像）中已正确安装并配置好了 FFmpeg 环境变量。
3.  **Uptime Kuma 的数据对接**：
    目前视觉前端已全部闭环完毕（连动画开关都有了）。后续实施对接时，只需将应用的 URL 或 ID 传给后端或 Uptime Kuma 接口，替换现有的 mock `appDots` 数据即可。
